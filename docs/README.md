# Ykamina.ru

## Локальная разработка

1. Поднимите PostgreSQL: `docker compose up -d postgres`.
2. Скопируйте `frontend/.env.example` в `frontend/.env.local`.
3. Запустите сайт: `cd frontend && npm run dev`.

## Запуск панели суперадминистратора

1. Примените все миграции: `cd frontend && npm run db:migrate`.
2. Задайте переменные `SUPER_ADMIN_EMAIL`, `SUPER_ADMIN_PASSWORD` (не менее 12 символов), при необходимости `SUPER_ADMIN_NAME` и `SUPER_ADMIN_PHONE`.
3. Создайте или обновите учётную запись: `npm run admin:create`.
4. Войдите на `/account/login`; после входа панель доступна по адресу `/admin`.

Для локального первого запуска без заранее заданных данных можно выполнить `npm run admin:create -- --generate`. Будет создан `admin@ykamina.ru` со случайным одноразовым паролем, который команда покажет один раз.

Панель защищена ролью `super_admin`. В ней доступны сводка состояния бизнеса, публикация каталога, управление ролями и блокировкой пользователей, состояние интеграции поставщика и аудит критичных изменений.

## Данные

SQL-схема и демонстрационные данные лежат в `database/`. Повторный seed можно выполнить командой `npm run db:seed` из `frontend`.

## Оплата через Т‑Банк

Примените миграции командой `npm run db:migrate`. Затем в `frontend/.env.local` задайте `TBANK_TERMINAL_KEY`, `TBANK_PASSWORD` и публичный адрес сайта в `NEXT_PUBLIC_SITE_URL`. Если пароль содержит `$`, экранируйте его как `\$`, чтобы загрузчик окружения не принял остаток строки за имя переменной. Секретный пароль используется только серверным API и не попадает в браузер. Для уведомлений банка адрес `/api/payments/tbank/notification` должен быть доступен из интернета по HTTPS; локальный `localhost` подходит только для проверки интерфейса.

Для онлайн-чека задайте `TBANK_TAXATION`, `TBANK_TAX`, `TBANK_FFD_VERSION` и `TBANK_PAYMENT_METHOD`. При создании платежа сайт передаёт в `Init` объект `Receipt`: контакт покупателя и позиции заказа с актуальными ценами, количеством, суммой, ставкой налога и единицей измерения. Тестовый терминал настроен на `usn_income`, `none`, ФФД 1.2 и `full_prepayment`; перед включением рабочего терминала эти значения необходимо сверить с системой налогообложения компании и настройками онлайн-кассы.

Отмена доступна суперадминистратору в разделе «Заказы и оплаты». Заказ без созданного платежа отменяется локально. Если у заказа есть `PaymentId`, после подтверждения выполняется полный возврат через метод Т‑Банка `Cancel`; повторная отмена блокируется, ответ банка и причина сохраняются в заказе, действие записывается в аудит. Для полного возврата объект `Receipt` в `Cancel` не передаётся: чек возврата формируется Т‑Банком автоматически на основании исходного чека. Частичный возврат пока не поддерживается.

## Импорт RealFlame

`https://realflame.ru/upload/partners.csv` — открытая партнёрская CSV-выгрузка. В ней есть название, артикул, цена, картинка, размеры и описание.

`https://realflame.ru/price/RealFlameStock.xlsx` содержит статусы доступности для Москвы и Санкт-Петербурга, а также оптовую и рекомендованную розничную цену. Точного числа единиц в файле нет, поэтому импортёр сохраняет фактические статусы (`Много`, `По запросу`) в `products.availability`, не подменяя их цифрами.

После запуска PostgreSQL выполните `npm run supplier:realflame:sync` из `frontend`. Команда загружает оба файла, определяет товар по артикулу, создаёт/обновляет карточку, цену, доступность и журнал запуска в `import_runs`. Для регулярного обновления добавьте эту команду в cron/планировщик раз в час. Не давайте доступ к импорт-эндпоинту из браузера — запуск должен оставаться серверной задачей.
